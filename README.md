# 🎙️ AI Ethics Podcast Resources

> **A research hub for the podcast episode: *"Who Gets to Define Bias? Inside the Open-Source Battleground of AI Fairness"***

---

## 🎧 The Podcast Angle

Technology and social justice have never been more entangled. As machine learning systems make consequential decisions — who gets a loan, who gets flagged by predictive policing, who receives a job interview — the question of *fairness* has become one of the most contested in modern software engineering.

But here's the twist: the fiercest debates aren't happening in academic journals or courtrooms. **They're happening in GitHub issue threads, pull request reviews, and open-source mailing lists** — written by the very engineers and researchers building the tools that millions will rely on.

This repository is the research backbone for a deep-dive podcast episode exploring:

- ⚖️ **The mathematics of fairness** — and why satisfying one fairness metric makes others mathematically impossible
- 🔬 **Who gets to define "bias"** — corporate labs, academic researchers, affected communities, or regulators?
- 🛠️ **The open-source fairness toolkit wars** — which algorithms get merged, which stay stale for years, and why
- 🌍 **The sociotechnical gap** — why a technically "fair" model can still cause real-world harm
- 🤖 **Individual vs. group fairness** — an unresolved philosophical schism baked into every toolkit

## 🤝 Contributing

This is a **living research document** — we want it to grow with the community.

### How to Contribute

1. **Add a resource**: Found a fairness toolkit, paper, dataset, or audit tool we missed? Open a PR adding it to `RESOURCES.md` with a one-line description and a link to the primary source.

2. **Surface a debate**: Spotted a heated GitHub issue thread, a stalled PR with philosophical disagreements, or a closed debate worth revisiting? Add it to `DEBATES.md` with:
   - A direct link to the thread
   - A neutral 2–3 sentence summary of the core disagreement
   - The names/handles of key participants (they're public contributors)
   - Your own take (clearly labeled as opinion)

3. **Correct the record**: If we've misrepresented a technical concept, a project's scope, or a contributor's position — please open an issue or PR immediately. Accuracy matters here.

4. **Start a discussion**: Open a GitHub Issue in this repo using the `[DISCUSSION]` tag. We actively read and may feature community debates in the podcast.

### Contribution Guidelines

- **Be specific**: Link to exact comments, not just issues. Quote directly when possible.
- **Be fair**: When summarizing debates, represent all sides. This is journalism, not advocacy.
- **Cite sources**: Academic papers, blog posts, and talks should include author, title, year, and URL.
- **No personal attacks**: Critique ideas, algorithms, and institutional choices — not individuals.
- **Disclose conflicts**: If you work for or are affiliated with a project you're writing about, say so.

### What We're Especially Looking For

- 🔍 Issues/PRs where maintainers *rejected* a fairness metric or algorithm and explained why
- 📊 Cases where the same dataset was used to argue for *opposite* fairness conclusions
- 🏛️ Examples of regulatory or legal pressure showing up in technical issue threads
- 🌐 Non-Western perspectives on fairness that challenge default assumptions in Western ML research
- 🧑‍🤝‍🧑 Evidence of (or absence of) affected community input in toolkit design decisions
